Главная идея: не «учить технологии» — делать продукты

К 2030 году ИИ пишет шаблонный код за секунды. Работодателю нужен человек, который берёт бизнес-задачу и доводит её до работающего сервиса в продакшене. Каждый год — не «выучить технологию», а сделать продукт.

Почему Python + Backend, с выходом в AI

Python — единственный язык, одинаково нужный в бэкенде, data-инжиниринге и AI. К 2030 году ожидается острый дефицит AI-специалистов: в России сейчас ~57 тыс. специалистов, нужно ещё 89 тыс. Ты выпускаешься прямо в этот дефицит.

🏗️ 1 курс — Фундамент (2026 – 2027)

Цель: написать первые рабочие приложения, не бояться кода

🍂 Осень

  • SoloLearn Python — до списков и функций. Stepik «Поколение Python»: if, for, while, задачи каждый день.
  • SQL с первого семестра: sql-ex.ru — SELECT, WHERE, GROUP BY. SQL учится быстро и нужен в любой нише.
  • Английский — 15 минут в день: читать документацию Python на английском. Не курсы — живая практика. Цель за 4 года: A2 → B2.
  • Pydroid на телефоне — задачи в очередях и транспорте.

🌱 Весна

  • Stepik: 50+ задач без подсказок — списки, строки, функции.
  • ООП + первый реальный проект: консольный менеджер задач с сохранением в SQLite. Не игрушка — рабочая программа.
  • Git: commit, push, pull, ветки — всё сразу на GitHub. Без Git прогресса нет.
  • SQL — продолжение: JOIN, подзапросы, агрегации на PostgreSQL.

☀️ Лето

  • Telegram-бот на aiogram — практика работы с внешним API и асинхронностью. Учебный проект.
  • Парсер на requests + BeautifulSoup — понять HTTP изнутри.
  • Первый REST API на FastAPI — небольшой: GET/POST, возврат JSON. Важнее бота для понимания бэкенда.
  • README к каждому проекту — писать для человека, который видит тебя впервые.
✅ Результат 1 курса 4 проекта на GitHub: CLI-менеджер, бот, парсер, первый FastAPI. Понимаешь SQL. Читаешь английскую документацию без переводчика.

🎓 2 курс — Специализация (2027 – 2028)

Цель: делать полноценные сервисы и получить стажировку

🍂 Осень

  • PostgreSQL глубже: индексы, транзакции, EXPLAIN ANALYZE.
  • Полноценный API на FastAPI или Django: авторизация (JWT), валидация (Pydantic), PostgreSQL через SQLAlchemy. CRUD с реальной логикой.
  • Git — командная работа: pull request, code review, rebase.
  • Английский: статьи dev.to и документацию фреймворков без переводчика.

🌱 Весна

  • PostgreSQL вручную: поднять без Docker — столкнуться с проблемами конфигурации.
  • Docker — после осознания зачем: когда почувствовал боль ручного деплоя — docker-compose становится решением, а не магией.
  • Задеплоить проект: Render или Railway — живая ссылка обязательна. Без деплоя проект не существует.
  • Резюме + GitHub-профиль: оформить, опубликовать на hh.ru.
📋 Критерии готовности к стажировке — проверь к февралю • GitHub: 4–5 проектов с README и живыми ссылками
• Один проект задеплоен и открывается по URL
• Резюме опубликовано на hh.ru
• 10–15 откликов каждую неделю — это отдельная задача, не «когда появится настроение»
• Целиться в продуктовые компании, не в аутсорс: там реальный code review и рост

☀️ Лето

  • Стажировка — нормально получить 30 отказов до первого оффера.
  • Если стажировки нет: крупный pet-проект на 2–3 месяца — что-то, чем реально можно пользоваться.
✅ Результат 2 курса Junior с портфолио. Понимаешь как работает сервис от HTTP-запроса до БД. Умеешь деплоить. Готов к собеседованиям.

📈 3 курс — Продукт с пользователями (2028 – 2029)

Цель: реальный сервис + коммерческий опыт + AI-интеграция

Главная задача курса Запустить сервис, которым пользуются реальные люди. Даже 20–50 пользователей — это принципиальная разница на собеседовании. Идеи: генератор конспектов, трекер расходов, сервис расписания, бот для одногруппников.

🍂 Осень

  • Работа / стажировка: 10–15 часов в неделю — реальный код, code review, дедлайны.
  • pytest: написать тесты к своим проектам. Без тестов нет доверия к коду.
  • Linux: терминал, SSH, nginx, systemd, логи, права доступа. Учить на живом сервере.
  • LeetCode — старт: 1–2 задачи в неделю. Только массивы и хэш-таблицы.
  • AI-интеграция — старт: вызвать API языковой модели (YandexGPT или актуальный в 2028). Обернуть в FastAPI-сервис.
📌 Почему AI-интеграция важнее классического ML К 2030 году работодатели спрашивают: умеешь ли ты интегрировать LLM в реальный сервис? Строить RAG? scikit-learn можно изучить попутно — ставить на него основной акцент не стоит.

🌱 Весна

  • Работа: уже 15+ часов в неделю.
  • Облака: Yandex Cloud или аналог — VPS, S3, базовый мониторинг.
  • CI/CD: GitHub Actions — автоматические тесты и деплой при пуше.
  • AI-интеграция — практика: RAG-сервис на своих данных. Инструменты оркестрации, актуальные на момент обучения.
  • LeetCode: Two Pointers и Sliding Window. 2–3 задачи в неделю.
  • Системный дизайн: читать реальные postmortem'ы (Cloudflare, Avito, Яндекс публикуют открыто). Разбирать одну архитектуру письменно в неделю — лучше YouTube-лекций.
  • Английский: tech-YouTube без субтитров. Цель: уверенный B1.

☀️ Лето

  • Коммерческий опыт: работать дальше.
  • LeetCode: Stack и Queue. К концу лета — 40+ решённых задач.
✅ Результат 3 курса Junior+. Реальный сервис с пользователями. Понимаешь AI-интеграцию на практике. Уверенный кандидат на backend-позицию.

🚀 4 курс — Выход на рынок (2029 – 2030)

Цель: масштабируемые системы, диплом как продукт, оффер

🍂 Осень

  • Работа: 12–15 часов в неделю — расти внутри хорошей компании.
  • Системный дизайн: архитектура сервисов, CAP-теорема, очереди, кэширование, балансировка.
  • LeetCode: Binary Search и деревья. К зиме — 70+ задач. Графы — последними.
  • AI-уровень: fine-tuning, agent-системы, оценка качества моделей. Не привязываться к конкретным библиотекам — ориентироваться на принципы.
  • Английский: уверенный B2 — GitHub Issues, RFC, конференции без субтитров.

🌱 Весна

  • Диплом — реальный продукт: API + БД + деплой + документация + тесты. Главный экспонат. Идеально — продолжение сервиса с пользователями из 3 курса.
  • Пробные собеседования: 5–10 интервью в компаниях куда не очень хочется — только тренировка.
  • Резюме: 2+ года опыта, живые ссылки, метрики: «сервис с 50 пользователями», «покрытие тестами 80%».

☀️ После выпуска

  • Полный рабочий день. Цель: сильный Junior+ или Middle — в зависимости от компании и темпа роста.
✅ Результат 4 курса Диплом + портфолио + 2+ года опыта + сервис с пользователями + понимание AI. Уровень сильного Junior+ или Middle.

Что делать, если отстал от плана

Жизнь почти никогда не идёт по идеальному плану. Это нормально — важно знать, как реагировать.

Не успел SQL к концу 1 курса
Закрыть пробел до следующего этапа. Не идти дальше с дырой в базе.
Нет стажировки к лету 2 курса
Крупный pet-проект + продолжать отклики осенью 3 курса.
Месяц без прогресса из-за сессии
Вернуться к последнему незакрытому шагу. Не пытаться догнать всё сразу.
Docker не понимается
Отложить. Сначала поднять PostgreSQL вручную — мотивация придёт сама.
LeetCode не идёт
Вернуться на категорию назад. Прогресс здесь нелинейный — это нормально.
Нет идей для проекта
Сделать то, что нужно тебе или однокурсникам. 20 пользователей достаточно.
💤 Про здоровье и долгосрочную дистанцию Четыре года — марафон, не спринт. Не жертвовать сном ради обучения: год стабильной работы по 1–2 часа в день полезнее месяца по 10 часов с последующим выгоранием. Отдых — часть плана.
